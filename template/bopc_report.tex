\section{Speed-up Analysis}

Two algorithms $A_1, A_2$ are given with the following parallel runtimes:
\[T_{par}^{A_1}(n,p) \in O(\frac{n \log n}{p} + \log^2(n)),\]
\[T_{par}^{A_2}(n,p) \in O(\frac{n \log n}{p} + 3\cdot n).\]

The sequential time for both is defined as $T_{seq}^*(n) \in O(n\log n)$.\\

The \textit{absolute speed-up $S_a$} is given by $S_a(n,p)= \frac{T_{seq}(n)}{T_{par}(n,p)}.$

For algorithm $A_1$, this gives:\\
\[S_a^{A_1}(n,p) = \frac{n \log(n)}{\frac{n \log n	}{p}+\log^2 n}.\]

As for algorithm $A_2$, this gives:\\
\[S_a^{A_2}(n,p) = \frac{n \log(n)}{\frac{n \log n	}{p}+3n}.\]

Table \ref{speedup_table} shows the different \textit{absolute speed-ups} for $n=500, p =\lbrace 4,16,64 \rbrace$.


		
\begin{table}[htb]
	\centering 
	\caption{\label{speedup_table}Absolute speed-up of algorithms with fixed n = 500 by number of cores.}
	\begin{tabular}{rrrr}
		\toprule
		 & p = 4 & p = 16 & p = 64 \\
		\midrule
		$S_a^{A_1}$ & 3.81 & 13.35 & 35.65  \\
		$S_a^{A_2}$ & 1.36 & 1.83 & 2.00  \\

	\end{tabular}
\end{table}

When comparing both algorithms with the ideal speed-up, $A_1$ performs nicely enough with 4 cores, moderately with 16 and bad with 64 cores. $A_2$ does not offer benefits of parallelization with any amount of cores. The absolute speed-up alone does not quantify the gain from parallelizing a code. If we look at the parallel efficiency is given by $E(n,p) = \frac{T_{seq(n)}}{p\cdot T_{par}(n,p)} = \frac{S_a(n,p)}{p}$, we can see that for both algorithms, the efficiency drops off quite quickly (see Figure \ref{figure:efficiency}). Table \ref{efficiency_table} shows the parallel efficiency of the algorithms. Both do not scale well.

The potential speed-up is determined by the sequential fraction of a code. \\

For $A_1$, the sequential fraction $s_{A1}$ is given by:\\

\[s_{A1}(n) = \frac{\log^2(n)}{n\log n} = \frac{\log n}{n}.\]\\

The potential speed-up for algorithm 1 is then given by\\
\[S^{A_1}(n,p) = \frac{1}{s_{A1} + \frac{1-s_{A1}}{p}} = \frac{1}{\frac{\log n}{n} + \frac{1-\frac{\log n}{n}}{p}}\leq \frac{n}{\log n} = S^{A_1}(n).\]\\


Regarding $A_2$, $s_{A2}$ is given by:\\
\[s_{A2}(n) = \frac{3\cdot n}{n\log n} = \frac{3}{\log n}.\]\\

The potential speed-up for algorithm 2 is then given by\\
\[S^{A_2}(n,p) = \frac{1}{s_{A2} + \frac{1-s_{A2}}{p}} = \frac{1}{\frac{3}{\log n} + \frac{1-\frac{3}{\log n}}{p}}\leq \frac{\log n}{3} = S^{A_2}(n).\]\\


\section{Weak Scaling Analysis}

In order to get the average work, we solve $w = \frac{n^3}{p}$ for $n$. 

\[w = \frac{n^3}{p} \Longleftrightarrow n = \sqrt[3]{wp} \]

For fixed input size $n = 1222$ with one processor, the input size varies with the number of cores as follows: 

\noindent $n = 1222, p=2, w = 1222^3 \Rightarrow n_2 = 1222\cdot 2^{1/3}$\\
$n = 1222, p=6, w = 1222^3 \Rightarrow n_6 = 1222\cdot 6^{1/3}$\\
